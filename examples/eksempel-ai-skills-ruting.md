# Eksempel: AI-SKILLS-ruting i et prosjekt

## Prosjekt

Initialisering av metode-repo for Iterativ AI-orkestrering.

## Mål

Gjøre et metodedokument om til et praktisk repo med forklaringer, prompts, maler, eksempler og Codex-instrukser.

## Valgte skills fra AI-SKILLS-SETS

| Skill | Nivå | Hvorfor valgt? | Lokal bruk |
|---|---|---|---|
| `mature-ai-skillset` | meta | Gir overordnet arbeidsform for AI: tydelig mål, begrensninger, ansvar og gjenbruk. | Brukes som prinsipp for hvordan ChatGPT og Codex skal arbeide. |
| `interview-driven-discovery` | core | Treffer metoden med intervjurunder som ekstraksjonsmotor. | Brukes til å forbedre intervjuprompt og intervjunotat. |
| `synthesis-and-distillation` | core | Treffer prompt-destillering og kondensering av lange samtaler. | Brukes til å forklare hvordan rå samtale blir operativ prompt. |
| `human-to-ai-translation` | core | Treffer behovet for å gjøre menneskelig metode AI-brukbar. | Brukes til å skille forklaring for mennesker fra instruks for AI. |
| `project-startup-guardrails` | core | Gir minimumsstruktur for nye prosjekter. | Brukes i standard prosjektflyt og problemnotat. |
| `source-backed-documentation` | core | Sikrer at viktige påstander kan spores til kilder. | Brukes til å behandle metodedokumentet som hovedkilde. |
| `success-feedback-loop` | core | Treffer tverrprosjektlig læring. | Brukes til reflection og AI-SKILLS-tilbakeføring. |
| `skillset-selection-and-routing` | meta | Forklarer hvordan skills bør velges uten å laste alt. | Brukes til dette rutingeksempelet og rutekort-malen. |

## Vurdert, men ikke valgt som standard

| Skill-type | Hvorfor ikke standard? |
|---|---|
| MCP-specialist | Nyttig bare hvis prosjektet faktisk handler om MCP. |
| Privacy operations specialist | Nyttig bare ved konkret personvern- eller behandlingsansvar. |
| FinOps og kommersiell styring | Relevant for kostnads- og leverandørprosjekter, ikke for alle metodeprosjekter. |
| Mobile, edge og offline | For smalt for standard prosjektflyt. |

## Hvordan dette påvirker repoet

Skillene brukes som inspirasjon til:

- bedre forklaring av intervjurunder
- tydeligere prompt-destillering
- en rutingmodell for hvilke skills som bør brukes når
- en regel om å ikke råimportere hele skill-biblioteket
- bedre reflection og tilbakeføring til AI-SKILLS

## Kort operativ regel

Start med få skills:

1. ett overordnet AI-atferdsrammeverk
2. to til fire core-skills som treffer prosjektfasen
3. en meta-skill hvis arbeidet handler om metodebasen
4. specialist-skills bare når domenet eksplisitt krever det

## Læringspunkt

AI-SKILLS-SETS er mest verdifullt når det brukes som rutet kunnskapsbase, ikke som rå dokumentdump.
