# Iterativ AI-orkestrering

Dette repoet forklarer en praktisk metode for å bruke ChatGPT, Codex, GitHub og AI-SKILLS sammen i prosjektarbeid.

Målet er enkelt: hvert prosjekt skal levere noe konkret, men også gjøre neste prosjekt smartere.

```text
Problemstilling
-> intervjurunder
-> prompt-destillering
-> ny operativ ChatGPT-chat
-> arkitektur, regler og struktur
-> Codex-handoff
-> GitHub som prosjektbase
-> reflection
-> AI-SKILLS-tilbakeføring
-> bedre startpunkt for neste prosjekt
```

## Hva metoden er

Iterativ AI-orkestrering er en arbeidsmåte der AI ikke brukes som en isolert svarmaskin, men som et sett med roller:

- ChatGPT brukes til drøfting, intervjuer, struktur, promptutvikling og reflection.
- Codex brukes til repo-arbeid, filer, kode, dokumentasjon og teknisk gjennomføring.
- GitHub brukes som stabil prosjektbase, historikk og sannhetspunkt.
- AI-SKILLS brukes som felles metodebase for prompts, regler, maler og læring.
- Metodeeier styrer retning, prioritering, kvalitet og tilbakeføring.

## Hvorfor repoet finnes

Uten struktur blir AI-arbeid fort engangschatter, løse prompts og dokumenter som ingen finner igjen. Dette repoet gjør metoden gjenbrukbar:

- nye prosjekter får en fast start
- intervjuer gir bedre kontekst
- prompts kan forbedres over tid
- Codex får tydeligere handoff
- GitHub holder filer og beslutninger samlet
- erfaringer blir konkrete AI-SKILLS-kandidater

## Start her

1. Les [hva metoden er](docs/01-hva-er-iterativ-ai-orkestrering.md).

3. Bruk [intervjustartprompten](prompts/01-intervjurunde-startprompt.md).

## Minste praktiske bruk

For et lite prosjekt holder dette:

1. Skriv et kort problemnotat.
2. Kjør en intervjurunde.
3. Be ChatGPT lage en destillert prompt.
4. Start ny chat med prompten.
5. Lag Codex-handoff.
6. La Codex oppdatere repoet.
7. Skriv reflection-notat.
8. Registrer minst ett mulig AI-SKILLS-forslag.

Det viktigste er ikke å fylle ut alt perfekt. Det viktigste er at prosjektet etterlater spor som neste prosjekt kan bruke.
