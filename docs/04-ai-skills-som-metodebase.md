# AI-SKILLS som metodebase

AI-SKILLS er den felles metodebasen som prosjektene bruker og forbedrer.

Den skal ikke være et lager for tilfeldige tips. Den skal samle prompts, regler, arbeidsflyter, maler og erfaringer som faktisk kan brukes igjen.

## Rollen til AI-SKILLS

| Rolle | Betydning |
|---|---|
| Metodebibliotek | Samler arbeidsmåter, prompts og regler. |
| Kompetanselager | Bevarer erfaringer som ellers ville forsvinne i enkeltsamtaler. |
| Startpunkt | Gir nye prosjekter bedre grunnstruktur. |
| Tilbakeføringspunkt | Tar imot læring fra ferdige prosjekter. |
| Kvalitetslag | Gjør gode praksiser felles, ikke private. |

## Hva som kan ligge i AI-SKILLS

- startprompts
- intervjuprompts
- Codex-handoff-mønstre
- reflection-regler
- dokumentasjonsmaler
- repo-konvensjoner
- kvalitetskriterier
- eksempler på gode prosjekter
- kjente feil og ting som bør unngås

## Relasjonen mellom prosjekt og AI-SKILLS

Hvert prosjekt bør svare på disse spørsmålene:

- Hvilke skills, prompts eller regler ble brukt?
- Hva fungerte godt?
- Hva fungerte dårlig?
- Hva ble endret lokalt i prosjektet?
- Hva bør løftes tilbake til AI-SKILLS?
- Hva bør testes i flere prosjekter før det gjøres til standard?

## Tilbakeføring må være konkret

Et godt AI-SKILLS-forslag skal si:

- hva som foreslås endret
- hvorfor det bør endres
- hvilken erfaring forslaget bygger på
- hvor i AI-SKILLS det kan høre hjemme
- hvordan forslaget kan testes

Ikke tilbakefør alt. Tilbakefør det som har gjenbruksverdi.

## Når kilden er et større skill-sett

Hvis et prosjekt bruker et større skill-bibliotek som `AI-SKILLS-SETS-main`, skal prosjektet ikke kopiere alt inn i metodebasen.

Bruk biblioteket som kilde for:

- ruting
- inspirasjon
- sammenligning
- eksempler
- kandidatforslag

Se `docs/11-ai-skills-set-som-kildemateriale.md` for anbefalt arbeidsmåte.
