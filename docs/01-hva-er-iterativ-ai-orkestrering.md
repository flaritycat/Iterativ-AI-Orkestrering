# Hva er Iterativ AI-orkestrering?

Iterativ AI-orkestrering er en metode for å koble ChatGPT, Codex, GitHub og AI-SKILLS i en gjentakende arbeidsflyt.

Metoden brukes når et prosjekt ikke bare skal produsere en leveranse, men også gi læring som forbedrer fremtidige prosjekter.

## Kort forklart

I vanlig AI-bruk starter man ofte med en prompt og får et svar. I denne metoden bygges det en arbeidsprosess rundt AI:

- ChatGPT hjelper med å utforske, intervjue, strukturere og evaluere.
- Codex gjennomfører repo-arbeid, filendringer, kode og dokumentasjon.
- GitHub samler prosjektets filer, historikk og beslutninger.
- AI-SKILLS tar vare på læring, prompts, regler og metoder som kan brukes på tvers.

Poenget er ikke at AI skal gjøre alt. Poenget er å gi hvert verktøy riktig rolle.

## Hovedformelen

```text
Prosjektarbeid
+ ChatGPT
+ Codex
+ GitHub
+ AI-SKILLS
+ reflection
= selvforbedrende prosjektarbeid
```

## Hva skiller metoden fra tilfeldig AI-bruk?

Tilfeldig AI-bruk gir ofte raske enkeltsvar. Iterativ AI-orkestrering lager spor:

- hva problemet var
- hvordan innsikt ble hentet ut
- hvilken prompt som ble destillert
- hvilke regler og valg som ble satt
- hva Codex fikk beskjed om å gjøre
- hva prosjektet lærte
- hva som bør tilbakeføres til AI-SKILLS

Dette gjør at neste prosjekt ikke starter på null.

## Når metoden passer

Bruk metoden når prosjektet har ett eller flere av disse behovene:

- uklart problem som må presiseres
- mye taus kunnskap hos metodeeier
- behov for dokumentasjon og sporbarhet
- overgang fra ChatGPT-drøfting til Codex-arbeid
- behov for gjenbrukbare prompts og maler
- læring som bør brukes i senere prosjekter

## Når metoden bør holdes lett

Ikke bruk full struktur hvis oppgaven er liten. For små oppgaver kan minimum være:

1. kort problemnotat
2. kort intervjurunde
3. destillert prompt
4. Codex-handoff
5. reflection med ett læringspunkt

Metoden skal hjelpe arbeidet, ikke bli arbeid i seg selv.
