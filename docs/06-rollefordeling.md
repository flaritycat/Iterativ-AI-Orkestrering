# Rollefordeling

Metoden fungerer best når hvert verktøy får en tydelig rolle.

## Oversikt

| Komponent | Primær rolle | Typisk bruk |
|---|---|---|
| ChatGPT | Refleksjonsrom og strukturverktøy. | Problemforståelse, intervjurunder, prompt-destillering, arkitekturdrøfting, evaluering. |
| Codex | Teknisk gjennomføringsverktøy. | Repo-arbeid, filer, kode, dokumentasjon, implementering og teknisk forbedring. |
| GitHub | Stabil prosjektbase. | Filstruktur, historikk, commits, issues, pull requests og dokumentasjon. |
| AI-SKILLS | Felles metodebase. | Gjenbrukbare prompts, regler, maler, reflection og læring. |
| Metodeeier | Retning og vurdering. | Prioritering, kvalitetskontroll, beslutninger og godkjenning. |

## ChatGPT

ChatGPT brukes tidlig og sent i prosessen.

Tidlig:

- utforske problem
- intervjue metodeeier
- finne begreper og avgrensninger
- destillere prompt

Sent:

- evaluere resultat
- identifisere forbedringer
- formulere AI-SKILLS-forslag

## Codex

Codex brukes når arbeidet skal inn i filer, repo og teknisk struktur.

Codex bør få:

- klart mål
- relevant filstruktur
- regler for hva som skal og ikke skal endres
- krav til verifikasjon
- forventet sluttstatus

## GitHub

GitHub er sannhetspunktet. Det som skal leve videre, bør inn i repoet.

Bruk GitHub til:

- README og docs
- promptbibliotek
- templates
- eksempler
- beslutningslogg
- historikk

## AI-SKILLS

AI-SKILLS er ikke der alt havner automatisk. Det er der modne, gjenbrukbare forbedringer havner etter reflection.

Bruk prosjektets `ai-skills/`-mappe som mellomstasjon før endringer løftes inn i felles metodebase.
