# Artefakter og dokumentasjon

Metoden skiller mellom prosjekt-output og metode-output.

Prosjekt-output er det prosjektet leverer. Metode-output er det prosjektet lærer.

## Minimumssett

| Artefakt | Formål |
|---|---|
| Problemnotat | Forklarer problem, mål, kontekst og avgrensning. |
| Intervjunotat | Samler innsikt fra intervjurunder. |
| Destillert prompt | Starter ny operativ ChatGPT-chat med ren kontekst. |
| Arkitektur- og regelnotat | Definerer struktur, valg og arbeidsregler. |
| Codex-prompt | Overfører arbeid til Codex. |
| Repository-dokumentasjon | Gjør repoet forståelig for nye lesere. |
| Beslutningslogg | Viser hva som er valgt, utsatt og endret. |
| Reflection-notat | Evaluerer resultat og arbeidsmåte. |
| AI-SKILLS-endringsforslag | Gjør læring overførbar. |

## God dokumentasjon er praktisk

Skriv så en ny person kan bruke materialet.

Unngå:

- lange bakgrunnstekster uten handling
- interne forkortelser uten forklaring
- beslutninger uten begrunnelse
- prompts uten bruksområde
- eksempler uten kontekst

Gjør heller dette:

- skriv kort hvorfor filen finnes
- vis når den brukes
- gi en konkret mal
- pek til neste steg

## Prosjektcase som læringsnode

Et prosjektcase skal vise både:

- hva prosjektet produserte
- hvordan metoden ble brukt
- hva prosjektet lærte
- hva som bør tilbakeføres

Dermed blir caset en node i læringssløyfen, ikke bare en rapport.
