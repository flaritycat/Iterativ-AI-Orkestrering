# Tverrprosjektlig læringssløyfe

Den tverrprosjektlige læringssløyfen er det som gjør metoden iterativ.

Et prosjekt bruker eksisterende metode, tester den i praksis og sender forbedringer tilbake. Neste prosjekt starter dermed med bedre prompts, bedre regler og bedre arbeidsflyt.

## Sløyfen

```text
AI-SKILLS
-> nytt prosjekt
-> ChatGPT-dialog
-> Codex og GitHub
-> reflection
-> endringsforslag
-> AI-SKILLS
-> neste prosjekt
```

## Tre nivåer

| Nivå | Hva utvikles | Hvordan læringen flyter |
|---|---|---|
| Prosjektnivå | Leveransen, repoet og dokumentasjonen. | ChatGPT, Codex og GitHub driver prosjektet fremover. |
| Ferdighetsnivå | Prompts, regler, maler og arbeidsmåter. | AI-SKILLS brukes, testes og forbedres. |
| Porteføljenivå | Mønstre på tvers av prosjekter. | Erfaringer fra ett prosjekt påvirker senere prosjekter. |

## Hva prosjektet må etterlate

For å bidra til læringssløyfen bør hvert prosjekt ha:

- prosjektcase
- beslutningslogg
- reflection-notat
- AI-SKILLS-endringsforslag
- kort oppsummering av hva som bør gjenbrukes

## Når læring ikke bør tilbakeføres

Ikke alt hører hjemme i felles metodebase.

Hold forslag tilbake hvis de er:

- for prosjektspesifikke
- uprøvde og uklare
- bare personlige preferanser uten praktisk effekt
- overlapp med eksisterende skill
- for tunge til å være nyttige

Bruk heller backlog hvis forslaget er lovende, men ikke modent.
