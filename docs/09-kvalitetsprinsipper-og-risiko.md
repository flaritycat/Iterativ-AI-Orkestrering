# Kvalitetsprinsipper og risiko

Kvalitet i denne metoden handler om tydelighet, gjenbruk og sporbar læring.

## Prinsipper

| Prinsipp | Praktisk betydning |
|---|---|
| Start med drøfting | Ikke be AI produsere endelig output før problemet er forstått. |
| Intervju før struktur | Hent ut kunnskap før den formaliseres. |
| Destiller kontekst | Gjør lang samtale om til presise prompts. |
| Skill chat-typer | Bruk ny chat når arbeidet trenger renere mandat. |
| Dokumenter før Codex | Codex bør få klare mål, regler og filkontekst. |
| Bruk GitHub som sannhetspunkt | Det som skal vare, skal inn i repo. |
| Bygg inn reflection | Evaluer resultat og metode. |
| Tilbakefør læring | Gode forbedringer skal vurderes for AI-SKILLS. |

## Typiske risikoer

| Risiko | Konsekvens | Tiltak |
|---|---|---|
| For rask produksjon | Output treffer ikke egentlig behov. | Kjør intervjurunde og problemnotat først. |
| For mye kontekststøy | Chatten blir uklar og tung. | Destiller prompt og start ny operativ chat. |
| Utydelig Codex-handoff | Repo-arbeidet blir tilfeldig. | Lag konkret Codex-prompt med filstruktur og krav. |
| Manglende tilbakeføring | Læring blir lokal og forsvinner. | Bruk AI-SKILLS-endringsforslag som fast avslutning. |
| Reflection uten handling | Innsikt endrer ingenting. | Knytt reflection til konkrete endringer eller backlog. |
| Metodeinflasjon | For mange regler gjør metoden tung. | Fjern eller forenkle regler som ikke gir verdi. |

## Kvalitetssjekk før prosjektet lukkes

- Er problemstillingen tydelig?
- Finnes destillert prompt?
- Finnes Codex-handoff hvis Codex ble brukt?
- Er repoet forståelig for en ny leser?
- Finnes reflection-notat?
- Er AI-SKILLS-kandidater vurdert?
- Er neste steg tydelig?
