# Intervjurunder som ekstraksjonsmotor

Intervjurunder er metoden for å hente ut informasjon, vurderinger, erfaringer og taus kunnskap fra metodeeier.

Dette er ofte den viktigste delen av arbeidsflyten. AI kan lage bedre struktur når den først får bedre råmateriale.

## Kjernefunksjon

```text
Taus kunnskap
-> spørsmål
-> presisering
-> strukturert innsikt
-> prompt
-> regler
-> dokumentasjon
-> AI-SKILLS-kandidat
```

## Hvorfor intervjurunder er krevende

- Uferdige tanker må formuleres.
- Skjulte antakelser kommer frem.
- Motstridende mål blir synlige.
- Metodeeier må velge hva som faktisk betyr mest.
- Det går saktere enn å be AI produsere med en gang.

Det er nettopp derfor runden er nyttig. Den gjør uklart materiale brukbart.

## Anbefalt intervjuprosess

| Runde | Formål | Typisk output |
|---|---|---|
| Åpen kartlegging | Få frem rå kontekst og intuisjon. | Rånotater og temaer. |
| Presisering | Gjøre uklare ord og behov konkrete. | Begreper, avgrensning og prioritering. |
| Motsetninger | Avdekke konflikter, risiko og vanskelige valg. | Risikobilde og beslutningspunkter. |
| Erfaringsoverføring | Koble prosjektet til tidligere arbeid og AI-SKILLS. | Gjenbrukbare mønstre og ting som bør unngås. |
| Strukturering | Omforme materialet til prosjektgrunnlag. | Prompt, regler og dokumentasjonsplan. |
| Validering | Sjekke at oppsummeringen treffer intensjonen. | Godkjent eller korrigert arbeidsgrunnlag. |

## Regler for gode intervjurunder

- Still ett spørsmål om gangen.
- Start bredt, men snevre inn.
- Spør etter eksempler når svaret er abstrakt.
- Spør hva som er viktigst når alt virker viktig.
- Spør hva som kan gå galt.
- Avslutt med oppsummering og korrigering.

## Output som skal tas videre

Etter intervjurunden bør materialet kondenseres til:

- problemformulering
- kontekstnotat
- destillert prompt
- regelsett
- beslutningslogg
- mulige AI-SKILLS-kandidater

Hvis intervjuet ikke gir slike artefakter, blir innsikten liggende igjen i chatten. Da taper metoden mye av verdien.
