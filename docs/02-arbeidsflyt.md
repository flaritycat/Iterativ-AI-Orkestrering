# Arbeidsflyt

Arbeidsflyten går fra uferdig problemforståelse til strukturert prosjekt, teknisk gjennomføring og metodeforbedring.

```text
1. Problemåpning
2. Utforskende dialog
3. Intervjurunder
4. Prompt-destillering
5. Ny operativ ChatGPT-chat
6. Arkitektur, regler og struktur
7. Codex-handoff
8. GitHub-arbeid
9. Reflection
10. AI-SKILLS-tilbakeføring
```

## 1. Problemåpning

Start med en kort beskrivelse av behovet.

Output:

- første problemformulering
- kjent kontekst
- hva som er uklart

## 2. Utforskende dialog

Bruk ChatGPT som samtalepartner. Målet er ikke å produsere endelig løsning, men å forstå hva prosjektet egentlig handler om.

Output:

- tydeligere begreper
- avgrensninger
- hypoteser
- risikoer

## 3. Intervjurunder

La ChatGPT stille ett spørsmål om gangen for å hente ut taus kunnskap, erfaringer, prioriteringer og kvalitetskriterier.

Output:

- intervjunotat
- problemnotat
- beslutningspunkter
- mulige AI-SKILLS-kandidater

## 4. Prompt-destillering

Komprimer samtale og intervjuer til en presis prompt for videre arbeid.

Output:

- rolle
- mål
- kontekst
- regler
- forventet output
- kvalitetskriterier

## 5. Ny operativ ChatGPT-chat

Lim den destillerte prompten inn i en ny chat. Dette gir renere kontekst og mindre støy fra den utforskende fasen.

Output:

- fokusert arbeidsrom
- mer presise drøftinger
- bedre grunnlag for arkitektur og Codex

## 6. Arkitektur, regler og struktur

Definer hvordan prosjektet skal organiseres før teknisk arbeid starter.

Output:

- repository-struktur
- dokumentasjonsregler
- beslutningslogg
- tekniske eller metodiske begrensninger

## 7. Codex-handoff

Lag en konkret prompt som forteller Codex hva som skal gjøres i repoet.

Output:

- Codex-instruks
- filstruktur
- krav til endringer
- verifikasjon
- reflection-krav

## 8. GitHub-arbeid

Bruk GitHub som stabil base for filer, historikk og videre arbeid.

Output:

- README
- dokumentasjon
- prompts
- maler
- eksempler
- commits og historikk

## 9. Reflection

Evaluer både resultat og arbeidsmåte.

Output:

- hva fungerte
- hva fungerte ikke
- hva bør endres
- hva bør testes videre

## 10. AI-SKILLS-tilbakeføring

Gjør læring om til konkrete forslag.

Output:

- nye promptmaler
- endrede regler
- forbedrede arbeidsflyter
- nye skill-kandidater
- backlog-elementer
