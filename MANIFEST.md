# Manifest

Dette repoet er ikke et dokumentarkiv. Det er en praktisk metodebase.

## Formål

Repoet skal hjelpe en ny bruker med å forstå og bruke Iterativ AI-orkestrering i faktiske prosjekter.

Metoden skal gjøre tre ting samtidig:

1. produsere en konkret prosjektleveranse
2. skape læring om arbeidsmåten
3. forbedre felles metode, prompts og AI-SKILLS for neste prosjekt

## Metodekjerne

Iterativ AI-orkestrering bygger på denne grunnideen:

```text
Prosjektarbeid + AI-verktøy + GitHub + AI-SKILLS + reflection
= et selvforbedrende prosjektøkosystem
```

## Prinsipper

- Start med drøfting, ikke produksjon.
- Bruk intervjurunder til å hente ut taus kunnskap.
- Destiller utforskende samtale til presise prompts.
- Skill mellom utforskende chat og operativ chat.
- Dokumenter arkitektur, regler og beslutninger før Codex-handoff.
- Bruk GitHub som sannhetspunkt.
- Evaluer både resultatet og arbeidsmåten.
- Tilbakefør læring til AI-SKILLS når den er konkret nok til gjenbruk.

## Hva som hører hjemme her

- forklarende metodefiler
- praktiske promptmaler
- prosjekt- og reflection-maler
- eksempler som viser bruk
- forslag til AI-SKILLS-forbedringer
- Codex-instrukser for videre arbeid
- visualiseringsnotater for presentasjon og formidling

## Hva som ikke hører hjemme her

- løse notater uten tydelig bruk
- akademisk fylltekst
- utestede regler som fremstår som fasit
- prosjektspesifikke detaljer uten læringsverdi
- prompts som ikke kan gjenbrukes eller forbedres

## Vedlikeholdsregel

Når et prosjekt er ferdig, skal repoet kunne svare på tre spørsmål:

1. Hva lærte prosjektet oss?
2. Hvilke deler av metoden bør endres?
3. Hvilke forslag bør løftes videre til AI-SKILLS?
