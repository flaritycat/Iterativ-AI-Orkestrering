# Prompt 04: Arkitektur og regler

## Formål

Bruk denne når prosjektet har nok kontekst til å definere struktur, regler, repo-logikk og beslutninger.

## Prompt

```text
Du skal hjelpe meg å definere arkitektur, regler og struktur for prosjektet.

Bruk prosjektkonteksten under og lag et praktisk grunnlag som senere kan brukes i Codex og GitHub.

Du skal produsere:

1. Anbefalt repository-struktur
2. Forklaring av hvorfor strukturen passer
3. Arbeidsregler for videre ChatGPT-arbeid
4. Arbeidsregler for Codex
5. Dokumentasjonsregler
6. Kvalitetskriterier
7. Risikoer og tiltak
8. Beslutninger som bør loggføres
9. Mulige AI-SKILLS-kandidater

Hold strukturen enkel. Ikke lag flere mapper eller regler enn prosjektet trenger.

Marker tydelig:

- beslutninger som virker modne
- antakelser som må valideres
- spørsmål som bør avklares før Codex-handoff

Prosjektkontekst:
[SETT INN KORT PROSJEKTKONTEKST HER]
```
